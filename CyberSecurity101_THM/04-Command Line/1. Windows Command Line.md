# Task 1 – Introduction

## Objective
Evaluate the operational advantages of Command Line Interfaces (CLIs) over Graphical User Interfaces (GUIs) and establish a secure terminal session with the target host.

## Key Concepts
- **CLI Efficiency**: Provides faster workflow execution, minimal system resource consumption, simple task automation via batch scripts, and efficient remote management over low-bandwidth channels.
- **Default Interpreter**: MS Windows utilizes `cmd.exe` as its default legacy command-line interpreter.
- **Remote Access Architecture**: Managing remote Windows instances via SSH provides a lightweight alternative to full RDP graphics streaming.

## Commands Used
```bash
# Establish remote SSH connection from AttackBox to target Windows VM
ssh user@<MACHINE_IP>
```

## What I Learned
- `cmd.exe` serves as the primary command interpreter across legacy and modern Windows installations.
- Command-line workflows bypass graphical desktop rendering overhead, making them essential for automated incident response and remote server triage.

# Task 2 – Basic System Information

## Objective
Extract detailed host hardware specifications, kernel build details, and global environment execution paths.

## Key Concepts
- **Environment Path Mapping**: Executable binaries can only be invoked directly without explicit file paths if their parent directory exists inside the system `%PATH%` variable.
- **Paging Command Output**: Piping dense terminal output into `more` forces page-by-page display, preventing output buffer flooding.
- **System Telemetry**: Obtaining OS build identifiers and hostnames provides immediate situational awareness during security assessments.

## Commands Used
```cmd
:: Display current environment variable configurations and system PATH
set

:: Query operating system kernel version details
ver

:: Extract comprehensive system information (OS name, build type, memory)
systeminfo

:: Display installed device drivers with page-by-page output
driverquery | more

:: Clear terminal screen buffer
cls

:: Display built-in command help manuals
help <command>
<command> /?
```

## What I Learned
- `ver` provides immediate OS kernel build numbers.
- `systeminfo` delivers a consolidated inventory of system architecture, hotfixes, network adapters, and domain/standalone status.
