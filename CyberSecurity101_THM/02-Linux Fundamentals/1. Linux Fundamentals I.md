# Task 6 – Searching for Files

## Objective

Automate file location searches and perform deep-text pattern matching queries across dense system asset paths.

## Key Concepts

- **Wildcard Lookups**: Utilizing symbol characters (`*`) to create dynamic search rules matching file segments or specific file extensions globally.
- **Recursive Content Scanning**: Forcing regex scanners to traverse downward through every lower folder line to discover target string elements across massive historical logs.

## Commands Used

```bash
# Locate specific files within the current pathway structure matching a naming string
find -name "*.txt"

# Search line logs inside a file to match and isolate a targeted string pattern
grep "TARGET_STRING" logfile.log

# Scan recursively through all child directories to uncover a string flag
grep -R "TARGET_STRING" /path/to/folder/
```

## What I Learned

- Manual exploration using `cd` and `ls` is highly inefficient; programmatic searching via `find` locates forgotten assets across entire filesystems instantly.
- The `grep` utility isolates small indicators of compromise (IoCs) or administrative parameters from thousands of dense system server logs.

# Task 7 – An Introduction to Shell Operators

## Objective

Implement stream management redirectors and terminal execution operators to chain multiple utilities together.

## Key Concepts

- **Background Process Optimization**: Shifting resource-heavy operations to the system background to maintain interactive access to the primary shell stream.
- **Conditional Chaining Logics**: Binding commands sequentially so secondary executions trigger exclusively if the primary task finishes successfully.
- **Destructive vs. Append Redirection**: Standard redirection overwrites file content entirely, whereas append redirection matches existing data lines and slots fresh logs straight onto the bottom boundary.

## Commands Used

```bash
# Execute a script or process asynchronously in the terminal background
command &

# Execute command2 sequentially only if command1 returns a successful status code
command1 && command2

# Redirect an output stream into a target file, overwriting existing file content
echo "string" > filename

# Append an output string directly onto the bottom line of a file without loss of existing data
echo "string" >> filename
```

## What I Learned

- Stream formatting tools allow text outputs to be dynamically converted straight into persistent log storage records.
- Chaining commands via logical operators streamlines administrative workflows by condensing multi-step tasks into single-line inputs.
