# Task 4 – The File System

## Objective

Examine the architectural design, access mechanics, and cryptographic attributes of the Windows file storage system.

## Key Concepts

- **Journaling Fault Tolerance**: NTFS uses automated transaction logging to dynamically repair disk structures following unexpected system crashes.
- **Granular Access Matrices**: The Security tab exposes explicit permissions (e.g., Full Control, Modify, Read & Execute) assigned to explicit groups or users.
- **Steganographic Evasion (ADS)**: Alternate Data Streams permit files to bind multiple raw `$DATA` streams, leaving secondary streams invisible to default file managers.

## What I Learned

- NTFS removes the 4GB file size constraints found in legacy formats like FAT32.
- Web browsers append distinct zone identifiers to an asset's alternate data stream to tag it as an untrusted Internet download.
- Security teams audit file properties via standard OS tabs to identify access configurations and potential permission inheritance issues.

## Objective

Locate critical system directories and utilize system variables to safely navigate underlying OS assets.

## Key Concepts

- **Path Environment Variable Mapping**: System environment variables hold global operating parameters, ensuring files can be referenced seamlessly regardless of their exact physical disk drive.
- **Core OS Integrity**: The `System32` directory functions as the central storage repository for critical executables, drivers, and libraries necessary to boot and run the OS.

# Task 7 – User Account Control

## Objective

Analyze the isolation mechanics of User Account Control (UAC) and its role in minimizing privilege escalation risks.

## Key Concepts

- **Least Privilege Enforcement**: Running an administrative account with standard non-elevated tokens prevents malware from automatically inheriting high-integrity execution bounds.
- **Security Boundary Demarcation**: Windows explicitly overlays a distinctive shield icon on installers or binaries that demand credential elevation or explicit administrative approval.

## What I Learned

- By default, standard User Account Control prompts do not apply to the built-in local administrator account.
- When an operation requires elevated tokens under a standard user account, UAC isolates the prompt and demands valid administrator credentials to proceed.
- If a UAC prompt is ignored or denied, Windows terminates the target installation stream to protect systemic integrity.
